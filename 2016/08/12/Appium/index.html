iOS自动化测试初探——Appium

我接触到的第二个UI自动化测试框架是Appium。它可以用来测试iOS系统或者Android系统的移动应用，是跨语言的，可以支持用Python、Ruby、Java、PHP、JavaScript语言编写测试代码。Appium iOS底层使用苹果官方提供的UIAutomation驱动,使用任何语言写测试用例最终都转化为UIAutomation对相关元素的操作。

搭建Appium环境比较折腾，用Python来写测试代码要与Python的unittest框架结合，因此想使用好Appium还是需要自己花一些功夫滴。感觉只是学到了皮毛。

一、Appium环境搭建（亲测有效!）

首先，要测试iOS设备的话，Mac机是必须的。

使用命令行安装Appium，需执行如下操作：

  • 安装homebrow
    在终端输入如下命令：ruby -e “$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)”
  • 安装node.js

brew install node

  • 安装Appium

npm install –g appium

  • 安装Appium Client

在https://github.com/appium/python-client 下载整个项目，下载后解压。

在终端进入python-client-master目录，在终端输入如下命令：

python setup.py install

  • 启动Appium

每次想要使用Appium做自动化测试前，首先要使用如下命令启动Appium：

Appium &

  • 安装doctor命令

npm install appium-doctor –g

*检查Appium依赖

appium-doctor

该命令检测Appium所需的依赖是否都安装，若有的没安装，按照终端提示可以直接安装。

此外，可以安装Appium.dmg，是GUI形式的Appium，好处是可以使用
Inspector直接看App的层级结构。Appium官网可以下载最新的Appium客户端，但下载需要翻墙。我是在百度网盘下载的最新版本Appium.dmg，地址为：http://pan.baidu.com/s/1jGvAISu 。

二．Appium原理

Appium采用C/S架构，客户端是一个扩展的Webdriver的库，服务端监听客户端发来来带有对移动设备操作的命令的http请求，并对移动设备发送命令，接受移动设备的执行结果，再把执行结果发送给客户端。

三．Python unittest框架

Python的unittest中，在我的代码中用到的有4个重要部分：test Case，test Suite，test Loader，test Runner。有关他们之间的关系先放一张静态类图：

1.各部分作用
从底向上分别说说每个部分的作用和用法：
TestCase:一个TestCase就是一个测试用例，包括测试环境搭建的setup方法、测试环境的清理的tearDown方法、和其他模拟用例测试操作的run方法。
写一个类，继承自unittest：TestCase

在这个类中，写setup方法，包括本机开辟的与Appium交互的端口信息和一些真机信息：

tearDown方法：

以及根据自己所测试App的测试用例实现的用户操作测试代码。
TestSuite：多个TestCase组合在一起构成TestSuite。
TestLoader：通过TestLoader，TestCase可以加载到TestSuite中。
TestRunner:执行测试用例run方法。
TestResult：保存测试结果。
写测试用例方法以test开头，这样在main函数中加载TestCase到TestSuite时，为每个TestCase自动创造一个TestCase对象。

四．写测试脚本

和UIAutomation一样，写一个测试用例也分为找到元素和对元素的操作两部分

  1. 定位元素：
    方法有：
    find_element_by_name 通过元素名定位
    find_element_by_xpath 通过XML Path定位
    通过元素名定位查找较快，但有的元素是没有名称的，所以不是所有元素适用。通过路径定位可以适用于所有场合，但查找的较慢。

  2. 操作元素
    方法有：
    click、tap 点击
    swipe滑动
    pinch 放大
    zoom 缩小
    所有API可以在https://testerhome.com/topics/3711 查到。
